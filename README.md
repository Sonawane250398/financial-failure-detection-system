# Financial Reporting Failure Detection System

End-to-end automated pipeline that detects financial reporting failures before they reach stakeholders.

80 exceptions caught across 1,000 transactions · 5 failure types · CRITICAL/HIGH/MEDIUM severity · zero manual steps · audit-ready CSV output

## Pipeline

Source Transactions → ETL Layer → SQL Reconciliation (5 checks) → Python Severity Engine → Exception Report
